TydiumCraft Information API


This page is intended for technical users who own a Geyser server.

What does it do?

The Information API helps you gather information on a Java or Bedrock player using their Server UUID. This information consist of the name, UUID, XUID, and linking data between Bedrock and Java. Every request returns with a JSON response.

Using the API!

Before you start using the API, make sure that you have the latest version of Floodgate! It is RECOMMENDED that you stay on the latest version of Floodgate! Then by using the UUID of any player (Not to be confused with XUID), you can put it into this link https://api.tydiumcraft.net/v2/players/info?uuid={UUID} and this will give you an output of the player data. If you wish to view an example request click here.

Key Definitions

  • "name" (String) - Name of the player you requested.
  • "platform" (String) - Returns Bedrock or Java.
  • "uuid" (String)- Returns the players server UUID.
  • "xuid" (Integer) - Returns the XUID of a Bedrock player (If Present).
  • "link" (Object/Boolean) - Returns the opposite platform information if the player is linked between Java and Bedrock. Returns false otherwise.
    • "linked" (Boolean) - Returns true if a player has linked accounts.
    • "java_name" (String) - Returns the name of the linked Java account.
    • "java_uuid" (String) - Returns the UUID of the linked Java account.
    • "bedrock_name" (String) - Returns the name of the linked Bedrock account.
    • "bedrock_xuid" (Integer) - Returns the XUID of the linked bedrock account.
    • "bedrock_uuid" (String) - Returns the Floodgate/Java server UUID of the linked bedrock account.

Notes

  • If "isLinked" returns "false" then linking data will automatically be hidden.
  • Data may appear differently when requesting with names, Bedrock UUIDs, or Java UUIDs
